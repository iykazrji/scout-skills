/scout Idea-to-Implementation Orchestrator

Takes a raw idea through interrogation, PRD creation, cost estimation, issue breakdown, and parallel execution — producing verified PRs.

Agent Roster

🟠 Scout Original Tab
Orchestrator — manages phases, state, handoffs, and user interaction across the entire pipeline.
🔴 Griller In-process
Interviews user via /grill-me protocol. Relentless technical interrogation until every decision branch is resolved.
🟡 Reconn cmux tab
Deep research agent — searches codebase, web, QMD, and claude-mem for context before each major phase.
🔵 Architect cmux tab
Writes comprehensive PRD via /write-a-prd and submits it as a GitHub issue.
💰 Bursar New In-process
Token cost estimator — analyzes PRD complexity, estimates implementation spend, and presents budget options before execution begins.
🟣 Slicer cmux tab
Breaks PRD into independently-grabbable vertical slice GitHub issues via /prd-to-issues.
🟢 Builder-N cmux tab ×N
Executes issues — one Builder per parallel issue. Creates branches, implements, verifies, and creates PRs.
🩷 Designer cmux tab
UI design orchestrator — dispatched for UI-heavy PRDs via /ui-designer. Produces design specs and mockups.

Bursar — Cost Estimation Engine

Token Baseline per Agent

Agent Baseline Tokens Notes
Slicer ~50,000 Reads PRD, explores codebase, creates issues
Reconn (per dispatch) ~30,000 Research before each wave
Builder (trivial) ~40,000 Config change, copy update
Builder (small) ~80,000 Single-file feature with tests
Builder (medium) ~150,000 Multi-file, exploration + testing
Builder (large) ~250,000 Cross-cutting, new patterns
Failure recovery (20%) ~60,000 Extra Reconn + retry per affected builder
Designer (if UI) ~100,000 Design specs + mockups
Inspector (if UI) ~40,000 Visual verification pass

User Options After Estimate

a) Proceed
Continue with current quality profile and full scope.
b) Scale Down
Reduce scope to a subset of requirements. User picks which features to keep.
c) Budget Mode
Switch to budget quality profile — agents use cheaper models.
d) Both
Scale down scope AND switch to budget mode for maximum savings.

Execution Modes

Interactive (default)

  • Grill interview
  • Grill approval gate
  • Bursar cost review
  • PRD approval
  • Issue breakdown approval
  • Wave approval
  • HITL pause

Yolo "shape it, ship it"

  • Grill interview
  • Grill approval (last stop)
  • Bursar: auto unless >$10
  • PRD auto-approved
  • Issues auto-approved
  • Waves auto-approved
  • HITL pause (needs judgment)

Auto fully autonomous

  • Grill skipped
  • No approval gates
  • Bursar: silent log only
  • PRD auto-approved
  • Issues auto-approved
  • Waves auto-approved
  • AFK (no pause)
